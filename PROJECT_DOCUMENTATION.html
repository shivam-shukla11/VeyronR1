1. Project Overview

VeyronR1 is a minimalist Laravel web application featuring user authentication (login and registration). It demonstrates the MVC (Model-View-Controller) architecture pattern used in modern web development.

Key Features:

  • User Registration - Create account with first name, last name, email, mobile, password
  • User Login - Login using email OR mobile number
  • Form Validation - Server-side and client-side validation
  • Password Security - Passwords are hashed using bcrypt
  • Session Management - Laravel handles user sessions automatically

MVC Explained:

  • Model - Represents data and database interactions (User.php)
  • View - HTML templates shown to users (.blade.php files)
  • Controller - Business logic that connects Model and View (AuthController.php)
How it works: When a user visits /login, Laravel checks routes/web.php to find which controller handles that URL. The controller then loads data from Models (if needed) and returns a View (HTML page).

Schema::create('users', function (Blueprint $table) {
    $table->id();                              // Auto-increment ID
    $table->string('first_name');              // First name column
    $table->string('last_name');               // Last name column
    $table->string('email')->unique();         // Unique email
    $table->string('mobile', 10)->unique();    // 10-digit mobile
    $table->string('password');                // Hashed password
    $table->string('role')->default('user');   // Default role
    $table->timestamps();                      // created_at, updated_at
});
Command to run migrations:
php artisan migrate
This reads all migration files and creates tables in the database.

5. Routes Explained

Routes define which URL shows which page. They connect URLs to Controllers.

use App\Http\Controllers\AuthController;

    return view('home');        // Shows home.blade.php

    ->name('login');            // GET: Shows login form

    ->name('register');         // GET: Shows register form

6. Controllers (Business Logic)

Controllers contain the logic that processes user requests. AuthController handles all authentication.

app/Http/Controllers/AuthController.php

6.1 Show Login Form

public function showLoginForm()
{
    return view('auth.login');  // Returns login.blade.php
}

Explanation: Simply returns the login view. view('auth.login') looks for the file at resources/views/auth/login.blade.php

6.2 Login Logic

public function login(Request $request)
{
    // Step 1: Validate input
    $request->validate([
        'identifier' => 'required',     // Email or mobile
        'password' => 'required',
        'role' => 'required|in:user,admin',
    ]);

    $identifier = $request->identifier;

    // Step 2: Find user by email OR mobile
    $user = User::where('role', 'user')
        ->where(function ($query) use ($identifier) {
            $query->where('email', $identifier)
                  ->orWhere('mobile', $identifier);
        })
        ->first();

    // Step 3: Verify password
    if ($user && Hash::check($request->password, $user->password)) {
        Auth::login($user);  // Create session
        return redirect()->route('home')
            ->with('success', 'Welcome back!');
    }

    // Step 4: Return error if invalid
    return back()->withErrors([
        'identifier' => 'Invalid email/mobile or password.'
    ]);
}

Code Breakdown:

  1. Validation: Ensures identifier and password are provided
  2. Database Query: Searches users table where email OR mobile matches input
  3. Password Check: Hash::check() compares input with hashed password
  4. Session: Auth::login() creates a session cookie
  5. Redirect: Sends user to home page with success message

6.3 Registration Logic

public function register(Request $request)
{
    // Step 1: Validate all fields
    $request->validate([
        'first_name' => 'required|string|max:255|regex:/^[a-zA-Z\s]+$/',
        'last_name' => 'required|string|max:255|regex:/^[a-zA-Z\s]+$/',
        'mobile' => 'required|string|digits:10|unique:users,mobile',
        'email' => 'required|email|unique:users,email',
        'password' => 'required|min:8|confirmed|regex:/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).+$/',
    ], [
        // Custom error messages
        'first_name.regex' => 'First name should only contain letters.',
        'mobile.digits' => 'Mobile number must be exactly 10 digits.',
        'password.regex' => 'Password must contain uppercase, lowercase & number.',
    ]);

    // Step 2: Create user in database
    $user = User::create([
        'first_name' => $request->first_name,
        'last_name' => $request->last_name,
        'mobile' => $request->mobile,
        'email' => $request->email,
        'password' => Hash::make($request->password),  // Hash password
        'role' => 'user',
    ]);

    // Step 3: Auto-login the new user
    Auth::login($user);

    // Step 4: Redirect with success
    return redirect()->route('home')
        ->with('success', 'Account created successfully!');
}

7. Models (Data Layer)

Models represent database tables and define what data can be saved.

app/Models/User.php 
<?php

namespace App\Models;

use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;

class User extends Authenticatable
{
    use Notifiable;

    // Fields that can be mass-assigned (saved to database)
    protected $fillable = [
        'first_name',
        'last_name', 
        'email',
        'mobile',
        'password',
        'role',
    ];

    // Fields hidden from JSON output (security)
    protected $hidden = [
        'password',
        'remember_token',
    ];

    // Custom attribute: Get full name
    public function getFullNameAttribute(): string
    {
        return "{$this->first_name} {$this->last_name}";
    }
}

Key Concepts:

  • $fillable: Only these fields can be saved using User::create()
  • $hidden: Password is never exposed in API responses
  • Accessor: getFullNameAttribute() lets you use $user->full_name

Important Elements:

  • @csrf: Required security token to prevent CSRF attacks
  • method="POST": Form submits data to server
  • action="{{ route('login') }}": Form submits to /login URL
  • {{ old('identifier') }}: Keeps entered value if validation fails
  • $errors->first(): Shows first validation error message

10. Security Features

10.1 Password Hashing

// When saving password:
'password' => Hash::make($request->password)

// Input: "MyPass123"
// Stored: "$2y$abcdefghij..."  (60+ characters)

// When verifying:
Hash::check($inputPassword, $storedHash)
// Returns true if match, false otherwise

Why? If database is hacked, real passwords aren't exposed.

10.2 CSRF Protection

<form method="POST">
    @csrf  <!-- Generates hidden token -->
    <!-- Rendered as: -->
    <input type="hidden" name="_token" value="random_token_here">
</form>

Why? Prevents malicious websites from submitting forms on behalf of users.

10.3 Validation

  • Server-side: Laravel validates all input before processing
  • Client-side: JavaScript validates before form submission
  • Unique constraint: Prevents duplicate emails/mobiles
  • Password rules: Enforces strong passwords

10.4 Mass Assignment Protection

// Only $fillable fields can be saved:
protected $fillable = ['first_name', 'last_name', 'email', ...];

// Prevents hackers from adding extra fields like:
// ?is_admin=true (won't work)
